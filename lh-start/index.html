Take a look at the 3 projects that LibreHealth hosts and get an idea of where you might want to contribute.

What is EHR?
According to Centers for Medicare and Medicaid Services, an electronic health record (EHR for short), is a digital report of a patient’s medical records managed by a hospital and can include “clinical data relevant to that person’s care under a particular provider.”
Us:
LibreHealth EHR is designed to be user-friendly, but also customizable and simplistic. The application is geared towards modern health-care professionals for day-to-day use.

What is Radiology?
Radiology is defined as the use of x-ray and radiation imaging for the purpose of arriving at a medical diagnosis.
Us:
LibreHealth Radiology is an extended version of LibreHealth Toolkit that seeks to integrate in Radiology Information Systems (RIS). The Radiology project is currently able to place radiology orders, define orderable imaging procedures, and create radiology reports.

What is Toolkit?
Toolkit is meant to function as the foundational API and data model for Health IT applications.
Us:
LibreHealth Toolkit is a fork of the OpenMRS platform but provides greater ease of use and installation along with performance enhancements. Toolkit can be used to build a customized LibreHealth EHR application specific to a institution's needs. It can be deployed in 8 languages and on Windows, Linux, and OSX.
